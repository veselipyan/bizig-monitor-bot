Все задачи трекаются в `cursor_temp/Bizig2Dev.md`:

**Типы задач:**

- **C{номер}** — CI/CD (GitHub Actions, автотесты)

**Статусы:**
```

[WIP] — в работе (указать: Agent, Files, Started)
[DONE] — завершено
[BLOCKED] — заблокировано (указать причину)
[REVIEW] — требует проверки
```

## 🎯 БАЗОВЫЕ ПРАВИЛА

### Перед началом работы:
1. **Прочитать** `cursor_temp/Bizig2Dev.md` — статус всех задач
2. **Прочитать** `cursor_temp/architecture.md` — структура проекта
3. **Проверить** нет ли заблокированных задач

### При взятии задачи в работу:
1. Обновить статус `[TODO]` → `[WIP]`
2. Указать `Agent: Bizik`, `Started: дата`, `Files: список`
3. Сохранить файл

### При завершении задачи:
1. Обновить статус `[WIP]` → `[DONE]`
2. Описать что сделано в Notes
3. Перенести в `cursor_temp/DEVELOPMENT_HISTORY.md` (если большая задача)

### Когда создавать отдельную задачу:
✅ Задача требует > 2-3 часов работы  
✅ Задача требует много токенов (сложная логика)  
✅ Задача затрагивает несколько модулей  
✅ Задача требует проектирования

❌ Мелкие правки (< 30 минут) — делаю сразу без задачи

## 🔥 GIT WORKFLOW

### Работа с Git:
```bash
# Клонирование репозитория
git clone <url>

# Создание ветки
git checkout -b feature/task-name

# Коммит
git add .
git commit -m "T{номер} — Описание изменений"

# Пуш
git push origin feature/task-name
```

### Правила:
- Коммитить **только когда явно попросят**
- Писать понятные commit messages (формат: `T{номер} — Описание`)
- Не делать force push без разрешения
- Работать в отдельных ветках для больших фич

### 🚫 ЗАПРЕЩЕНО:
- Force push в main/master
- Коммитить секреты (.env, токены)
- Обновлять git config без разрешения
